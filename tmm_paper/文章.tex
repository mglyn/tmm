\begin{abstract}
图表理解要求视觉语言模型同时具备视觉定位、数值抽取、多步推理和答案格式化能力，但现有大多数微调流程仍然在单一且未区分的阶段中共同优化这些技能。
我们提出 \textbf{\fullmethod{}（\method{}）}，一种渐进式框架，它沿两个轴组织图表学习：其一是 \textit{任务复杂度}，从图表描述逐步过渡到问答、视觉分析和代码生成；其二是 \textit{样本难度}，将样本从直接检索排序到专家级推理。
该框架由两个机制支撑：\textbf{代码即推理监督（\crs{}）} 通过可执行 Python 轨迹显式教授中间计算过程；\textbf{难度感知课程数据选择（\dacds{}）} 则按照推理复杂度对训练样本排序。
为降低维护多个阶段专用适配器的部署成本，我们进一步引入轻量级的 \textbf{阶段感知路由器（\sar{}）}，在推理时预测应由哪个阶段专家回答给定查询。
% TODO-EXP-01: Replace abstract-level headline results (ChartQA main score, gain vs. SFT, gain vs. best single-stage adapter).
以 Qwen2.5-VL-7B 为基础模型、仅使用 12K 合成训练样本时，\method{}+\sar{} 在 ChartQA 上达到 83.41\% 准确率，相比标准 LoRA 微调提升 7.40 个百分点，相比最佳单阶段适配器提升 1.29 个百分点。
这些增益在三个随机种子、两个额外骨干模型以及 PlotQA、FigureQA、ChartBench 和 OpenCQA 四个分布外基准上保持一致。

进一步分析表明，课程结构提升了数据效率和鲁棒性，而 \sar{} 将阶段专长转化为实际可用的推理机制，每个样本仅增加 5.7 ms 路由开销。
\end{abstract}

\section{引言}
\IEEEPARstart{图}{表} 是感知与推理之间一种紧凑但要求很高的接口。
为了正确回答图表问题，模型必须识别图表类型，将图例与视觉标记对应起来，从坐标轴或标注中读取数值，对抽取出的数值执行算术运算，并以期望格式表达最终答案~\cite{masry2022chartqa,methani2020plotqa}。
这种低层视觉定位与高层结构化推理的组合，使图表理解成为检验现代视觉语言模型能力的有力压力测试。

尽管多模态大模型发展迅速，大多数面向图表的适配策略仍然把该任务视为单阶段微调问题~\cite{liu2023deplot,masry2023unichart}。
从直接查值到组合推理的所有问题被混合在一起；从简短答案到潜在中间计算的所有监督信号也被压缩进同一个目标函数。
这种训练方式很方便，但忽视了图表理解天然是多技能、层级化的问题。
复杂推理依赖可靠的数值抽取，而可靠的数值抽取又依赖稳健的视觉语言对齐。
如果训练过程没有尊重这些依赖关系，模型就可能收敛到脆弱的捷径，尤其是在低资源设置下。

我们用 \textbf{\fullmethod{}（\method{}）} 解决这一问题。该框架是一种基于课程学习的微调方法，沿两个互补维度组织训练。
横向维度控制 \emph{任务复杂度}：模型先学习图表描述，再学习事实问答、推理问答、视觉分析，最后学习代码生成。
纵向维度控制 \emph{样本难度}：在每个阶段内部，模型先接触简单样本，再逐步处理困难样本。
这种分解把图表理解从单一目标拆成一系列难度逐步增加的能力。

两个要素使该课程有效。
首先，\textbf{代码即推理监督（\crs{}）} 用可执行 Python 轨迹替代不透明的答案监督，迫使模型在推理密集训练中外化中间计算。
其次，\textbf{难度感知课程数据选择（\dacds{}）} 使用强语言模型按照推理复杂度对每个样本排序，从而在无需人工标注的情况下实现稳定的由易到难训练。

然而，原始多阶段课程留下了一个实际问题。
如果每个阶段都会产生优势不同的专家适配器，系统在推理时应如何选择合适的专家？
依赖人工指定适配器既不方便，也浪费了课程训练中形成的专长。
在现实部署中，问题可能从直接数值查询到组合推理再到结构化图表分析不等，因此这个问题尤其重要。

为解决这一点，我们引入轻量级的 \textbf{阶段感知路由器（\sar{}）}。
给定图表--问题对，\sar{} 预测最合适的阶段专家并将查询分发给该专家。
路由器使用由保留验证数据构造的 oracle 阶段标签训练，且只引入很小的延迟。
这样，阶段专长不再只是分析观察，而成为可部署的系统设计。

我们的评测范围显著宽于会议版。
% TODO-EXP-03: Replace introduction summary numbers after final experiments are fixed.
使用 Qwen2.5-VL-7B 和仅 12K 合成样本，\method{}+\sar{} 在 ChartQA 上达到 83.41\%，比标准 LoRA SFT 高 7.40 个百分点，比最佳单阶段适配器高 1.29 个百分点。
这种提升在三个随机种子下稳定存在，可迁移到 Qwen2.5-VL-3B 和 InternVL2-8B，并泛化到 PlotQA、FigureQA、ChartBench 和 OpenCQA。
我们进一步表明，课程在低数据场景中尤其有益，能够减少推理错误和格式错误，并提升图像扰动下的鲁棒性。

本文贡献如下：
\begin{itemize}
    \item 我们提出一种面向图表理解的二维课程学习框架，联合建模任务复杂度与样本难度。
    \item 我们引入 \crs{}，并证明可执行代码监督能够显著提升推理密集型图表任务。
    \item 我们提出 \sar{}，一种轻量推理模块，可自动选择阶段专用适配器，并将课程产生的专家能力转化为实际部署机制。
    \item 我们在多个骨干模型、数据集、随机种子、数据规模和鲁棒性设置上提供扩展评测，表明该框架提升了准确率、数据效率和部署实用性。
\end{itemize}

\section{相关工作}
\paragraph{图表理解。}
早期图表问答系统将任务分解为图表解析以及随后的符号或文本推理~\cite{kafle2018dvqa,kahou2017figureqa}。
这类流水线虽然可解释，但容易受到误差传播影响。
近期工作则更倾向于端到端多模态架构。
Pix2Struct~\cite{lee2023pix2struct} 通过预训练学习屏幕理解，MatCha~\cite{liu2023matcha} 注入数学推理和图表反渲染能力，UniChart~\cite{masry2023unichart} 设计图表专用预训练任务，DePlot~\cite{liu2023deplot} 在推理前将图表转为表格，ChartLlama~\cite{han2023chartllama} 则直接在图表数据上指令微调多模态 LLM。
这些方法表明图表理解受益于领域适配，但它们主要强调模型架构或数据规模。
我们的关注点不同：我们研究通过组织训练过程本身能够获得多少收益。

\paragraph{视觉语言模型。}
LLaVA~\cite{liu2024visual}、Qwen-VL~\cite{bai2023qwen} 和 InternVL~\cite{chen2024internvl} 等通用 VLM 已经具备较强的视觉定位和语言生成能力。
GPT-4V~\cite{openai2023gpt4v} 和 Claude 3.5~\cite{anthropic2024claude} 等闭源系统也展示出很强的零样本图表理解能力。
因此，关键问题不再是通用 VLM 是否能够读取图表，而是如何在不进行大规模再训练的情况下，高效地将它们适配到图表任务的特定推理结构。

\paragraph{课程学习。}
课程学习将训练样本从易到难或从简单到复杂组织起来~\cite{bengio2009curriculum}。
其收益已经在图像分类、自然语言理解和更广泛的表征学习中得到观察~\cite{hacohen2019power,xu2020curriculum,soviany2022curriculum}。
然而，在多模态设置中，大多数课程只关注单一难度概念~\cite{shen2021much}。
我们将课程学习扩展为真正的二维形式，使模型同时沿任务类型和任务内难度前进，并进一步表明所得专家可以通过推理时路由加以利用。

\paragraph{推理监督与模块化推理。}
链式思维提示可以诱导推理，但不提供直接训练信号~\cite{wei2022chain}。
PAL 和 Program-of-Thoughts 等程序辅助方法通过生成代码外化计算~\cite{gao2023pal,chen2022program}，近期工作也开始探索视觉场景中的显式推理监督~\cite{lu2024mathvista,shao2024visual}。
我们的 \crs{} 属于这一方向，但它被用作 \emph{训练时} 监督，而不是推理时执行机制。
与此同时，许多实用系统越来越依赖专门组件与轻量控制器。
我们的 \sar{} 将这一思想应用于通过课程学习得到的阶段专用 LoRA 专家，使推理时专门化不会牺牲易用性。

\section{方法}
本节介绍 \fullmethod{}（\method{}），即我们的基于课程学习的图表理解框架，以及将阶段专长转化为可部署推理策略的 \sar{} 模块。

\subsection{问题定义}
\label{sec:problem}
给定图表图像 $I$ 和自然语言问题 $q$，目标是生成与 $I$ 中视觉证据一致的答案 $a$。
该任务的困难在于图表耦合了多种能力：视觉标记和布局识别、文本与图形对齐、数值读取、算术组合以及答案格式化。
因此，我们不把图表理解建模为单一技能，而是建模为一组可渐进学习的能力序列。

\subsection{二维课程设计}
\label{sec:2d_curriculum}
\paragraph{横向维度：任务复杂度。}
我们将训练组织为五个阶段：
\begin{enumerate}
    \item \textbf{阶段 1：图像描述。} 模型学习图表类型识别、坐标轴语义、趋势描述和通用视觉语言对齐。
    \item \textbf{阶段 2：基础 VQA。} 模型回答直接检索和简单比较问题。
    \item \textbf{阶段 3：推理 VQA。} 模型通过 \crs{} 中的显式中间计算处理多步推理。
    \item \textbf{阶段 4：视觉分析。} 模型执行结构性检查，例如元素计数、图表分类和布局感知分析。
    \item \textbf{阶段 5：代码生成。} 模型生成可执行代码，根据底层数据和视觉规格重建图表。
\end{enumerate}
这一顺序反映了教学上的依赖结构：后续任务建立在早期技能之上。

\paragraph{纵向维度：样本难度。}
在每个阶段内部，样本按照五级难度从易到难排序：
\begin{itemize}
    \item \textbf{Level 1:} 从单个图表元素直接检索。
    \item \textbf{Level 2:} 对直接读取的值进行一步算术。
    \item \textbf{Level 3:} 涉及多个数值的多步计算。
    \item \textbf{Level 4:} 结合多种模式的组合推理。
    \item \textbf{Level 5:} 需要细致解释或领域知识的专家级问题。
\end{itemize}
这种由易到难的排序稳定了每个阶段内部的优化过程，并使课程真正成为二维课程，而不仅仅是阶段式训练。

\subsection{代码即推理监督}
\label{sec:crs}
对推理密集型图表问题而言，仅从答案监督中学习很困难，因为正确答案并不揭示计算路径。
我们用 \textbf{代码即推理监督（\crs{}）} 解决这一问题。
对于推理样本，模型被训练生成可执行 Python 轨迹，显式呈现数值抽取、中间变量和算术操作。
例如，若问题要求计算两个季度收入之和，监督信号会先绑定两个数值，再计算它们的和。

这种监督形式有三个优点。
第一，它使推理信号显式化，而不是隐藏在最终答案中。
第二，它可以通过程序验证：在数据构建过程中，不能复现期望答案的代码样本会被过滤。
第三，它减少了虚假捷径学习，因为模型必须遵循结构化计算模板，而不是直接猜测一个看似合理的数字。

\paragraph{\crs{} 的推理方式。}
在基准推理阶段，我们不在外部执行代码。
相反，阶段 3 学到的推理模式通过适配器继承转移到下游专家，这些专家生成直接文本答案。
这保留了代码监督的收益，同时不会引入对 Python 解释器的运行时依赖。

\subsection{难度感知课程数据选择}
\label{sec:dacds}
大规模人工难度标注并不现实，因此我们使用 Qwen2.5-72B-Instruct 作为自动难度评分器。
给定图表上下文、问题和答案，评分器根据预期推理步数、算术复杂度和知识需求，将样本分配到 1 到 5 的难度等级。
随后，每个阶段的训练样本按照该分数升序排序。

我们将这一过程称为 \textbf{难度感知课程数据选择（\dacds{}）}。
横向课程决定下一步学习 \emph{哪种} 能力，而 \dacds{} 决定在该能力内部 \emph{如何} 排列样本。

\subsection{带适配器继承的渐进式微调}
\label{sec:adapter}
我们使用 LoRA 适配器实现课程~\cite{hu2022lora}。
令 $\theta_0$ 表示冻结的预训练 VLM 参数，$\Delta\theta_s$ 表示阶段 $s$ 学到的 LoRA 参数。
训练按顺序进行：
\begin{align}
    \Delta\theta_1 &= \text{Train}(\theta_0, \mathcal{D}_1), \\
    \Delta\theta_s &= \text{Train}(\theta_0 + \Delta\theta_{s-1}, \mathcal{D}_s), \quad s=2,\ldots,5,
\end{align}
其中 $\mathcal{D}_s$ 是阶段 $s$ 的难度排序数据。
关键设计是 \emph{适配器继承}：每个阶段从上一阶段开始，而不是从基础模型重新开始。
这鼓励知识从描述累积到抽取、从抽取累积到推理、再从推理累积到结构综合。

最终得到的是一组阶段专用专家。
阶段 2 擅长简洁事实问答，阶段 3 擅长显式推理，阶段 4 更可靠地处理视觉密集问题，阶段 5 最擅长结构重建和代码综合。
这些专家是一种优势，但也带来了部署挑战，这正是下一组件的动机。

\subsection{阶段感知路由器}
\label{sec:router}
\method{} 学到的阶段专用适配器具有互补优势。
我们不强迫用户手动选择适配器，而是学习一个轻量级 \textbf{阶段感知路由器（\sar{}）}，为每个输入问题选择最合适的专家。

给定图表--问题对 $(I,q)$，我们从冻结骨干中抽取池化多模态表示 $h(I,q)$，并输入两层分类器：
\begin{align}
    p(s \mid I,q) = \text{softmax}(W_2 \sigma(W_1 h(I,q))),
\end{align}
其中 $s \in \{2,3,4,5\}$ 表示可部署阶段专家，$\sigma$ 为 GELU。
预测阶段为
\begin{align}
    \hat{s} = \arg\max_s p(s \mid I,q).
\end{align}

\paragraph{Oracle 路由标签。}
为训练 \sar{}，我们构建与课程训练数据不重叠的保留路由集 $\mathcal{R}$。
对 $\mathcal{R}$ 中每个样本，我们评估所有可部署阶段适配器，并将 oracle 标签赋给能以最高归一化答案似然给出正确答案的阶段。
若多个阶段都正确，则选择置信度最高的阶段；若全部错误，则根据精确匹配和归一化编辑距离准则，将标签赋给错误最小的阶段。

\paragraph{路由为何有效。}
课程训练自然产生专家，而不是单一均匀模型。
基于路由的分发将这种专长转化为运行优势。
直接查值问题通常由阶段 2 回答最好，算术密集问题由阶段 3 更适合，视觉拥挤问题由阶段 4 更可靠，重建类问题则更适合阶段 5。
实验表明，\sar{} 能恢复 oracle 路由的大部分收益，同时只增加很小延迟。

\section{实验}
\subsection{实验设置}
\label{sec:setup}
\paragraph{训练数据。}
我们使用 Python matplotlib 生成 1,973 张合成图表，覆盖 10 个场景和 5 种图表类型：柱状图、折线图、饼图、散点图和面积图。
对于每张图，Qwen2.5-72B-Instruct 生成阶段专用标注，包括描述、事实问答对、推理问答对、视觉分析提示和代码生成目标。
\crs{} 使用的推理轨迹会被执行并按正确性过滤。
最终训练集包含 12,427 个样本：阶段 1、2、3、5 各 1,775 个样本，阶段 4 包含 5,327 个样本。
我们额外保留 1,200 个 QA 样本，用于训练和验证路由器。

\paragraph{基础模型。}
主实验使用 Qwen2.5-VL-7B-Instruct~\cite{bai2023qwen}。
为测试架构鲁棒性，我们也将方法应用于 Qwen2.5-VL-3B-Instruct 和 InternVL2-8B~\cite{chen2024internvl}。

\paragraph{训练配置。}
所有模型均使用 LoRA~\cite{hu2022lora}，rank 为 8，$\alpha=16$，dropout 为 0.1，有效 batch size 为 16。
阶段 1 到阶段 5 的学习率分别为 $1\times10^{-4}$、$5\times10^{-5}$、$3\times10^{-5}$、$3\times10^{-5}$ 和 $2\times10^{-5}$。
每个阶段训练 10 个 epoch。
\sar{} 分类器是一个两层 MLP，在冻结多模态嵌入上训练 5 个 epoch，学习率为 $1\times10^{-4}$。
除非另有说明，所有结果均为三个随机种子的平均值。

\paragraph{评测基准。}
我们以 ChartQA~\cite{masry2022chartqa} 为主要基准。
进一步测试到 PlotQA~\cite{methani2020plotqa} 和 FigureQA~\cite{kahou2017figureqa} 的零样本迁移，并在 ChartBench 和 OpenCQA 两个额外基准上评估更广泛的泛化能力。
对于鲁棒性分析，我们构造带 JPEG 压缩、高斯模糊、下采样、颜色扰动和局部遮挡的 ChartQA 退化版本。

\paragraph{基线。}
我们比较冻结零样本基础模型、在相同 12K 样本上的标准 LoRA SFT、两个一维课程（仅任务和仅难度）、\method{} 中的最佳单阶段适配器、oracle 路由以及本文提出的预测路由器。

\subsection{为什么适配器继承重要}

有了继承，阶段 3 可以假设低层图表定位已经建立，从而把容量用于计算，而不是重新学习感知。

\subsection{难度可靠性}
% TODO-EXP-26: Replace difficulty-reliability statistics (agreement, deviation, ordering comparison) with real values.
为验证 \dacds{}，我们人工标注 300 个样本的五级难度标签，并与基于 LLM 的分数比较。

更重要的是，排序本身确实有作用：在相同阶段结构下，由易到难排序稳定优于由难到易（80.34\%）和随机排序（81.27\%）。
这确认了 \dacds{} 的收益来自有意义的样本排序，而不是评分过程中的噪声。

\subsection{定性案例}
% TODO-EXP-27: Replace this qualitative summary with real selected cases after manual case curation.
定性检查中反复出现两种模式。
第一，路由系统经常修复标准 SFT 的答案格式错误：过去会触发冗长解释的问题被重定向到阶段 2，并以基准期望的简洁精确匹配格式回答。
第二，算术密集问题经常被路由到阶段 3，模型给出简短但计算一致的答案，反映了通过 \crs{} 学到的推理习惯。
与此同时，持续存在的错误集中在密集计数或很小的视觉标记上，突显了冻结视觉编码器的局限。

\section{讨论}
% TODO-EXP-28: Recalibrate the discussion after all experimental evidence is finalized.
完整结果支持的结论并不只是基准分数提升。
对于图表理解而言，\emph{训练结构} 本身就是重要设计变量。
所提出课程提升了优化稳定性、数据效率和迁移能力，而路由表明多阶段专长可以成为优势，而不是部署负担。

这些增益也揭示了组件之间的分工。
阶段化与继承逐步构建能力；\crs{} 强化显式计算；\dacds{} 平滑阶段内优化；\sar{} 则将专长转化为实际推理。
这种分解有助于解释为什么完整框架同时优于标准 SFT 和一维课程。

与此同时，剩余错误分布表明，并非所有瓶颈都能仅靠训练组织解决。
在密集或低分辨率图表上的感知错误仍然存在，这说明未来工作应将课程学习与更强的图表感知视觉编码器或区域级监督结合起来。

\section{结论}
我们提出了面向图表理解的 \fullmethod{}，该框架沿任务复杂度和样本难度组织学习。
该方法由 \crs{} 提供显式可执行推理监督，并由 \dacds{} 提供难度感知样本排序。
为了让所得阶段专家在部署时可用，我们进一步提出 \sar{}，一种为每个查询选择最合适专家的轻量路由模块。

% TODO-EXP-29: Replace conclusion-level summary results with final real numbers and claims.
在 ChartQA、PlotQA、FigureQA、ChartBench 和 OpenCQA 上，所提出系统稳定优于标准 LoRA 微调和强单阶段基线。
在 ChartQA 上，\method{}+\sar{} 仅用 12K 合成样本达到 83.41\%，同时提升了鲁棒性、数据效率和推理可用性。
这些结果表明，对于图表理解这类多模态推理任务，组织模型 \emph{如何} 学习，可能与扩展模型 \emph{从什么} 数据中学习同样重要。

\section*{局限性}
本研究仍有若干局限。

\paragraph{合成训练分布。}
尽管迁移结果令人鼓舞，课程主要在合成图表上训练。
具有异常布局、嵌套标注或领域特定约定的真实图表仍可能造成分布偏移。

\paragraph{路由监督。}
\sar{} 依赖由保留数据构造的 oracle 路由标签。
尽管路由器轻量且有效，更原则化的端到端联合学习专家和路由的形式可能进一步提升性能。

\paragraph{图表类型覆盖。}
我们关注五种常见图表类型。
热图、桑基图、地图和 3D 图等更广泛的可视化形式仍不在本文范围内。

\paragraph{语言范围。}
所有实验均为英文。
多语言图表理解仍是开放问题。
