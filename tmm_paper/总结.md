# TMM 投稿相对 ACM MM 原稿的新增工作与结果总结

## 一、新增核心方法：Stage-Aware Router (SAR)

ACM MM 原稿在 Method 部分结尾停在 "five specialized adapters can be deployed independently"，但没有解决**推理时到底该选哪个 adapter** 的问题。

TMM 稿新增 `§2.5 Stage-Aware Router`，包括：
- **SAR 模块设计**：冻结 backbone 抽取 multimodal embedding → 两层 MLP → softmax 输出 stage 概率
- **Oracle Routing 标签构造**：在 held-out routing set 上评估所有 deployable stage，用 correctness + 规则化 tie-breaking 分配 oracle label
- **路由训练与推理**：5 epoch 训练，推理时 cached-expert pipeline 下 dispatch

**结果**
| 指标 | 值 |
|---|---|
| SAR 预测路由 | **82.60%** |
| Best single-stage | 82.12% |
| Oracle routing | 87.48% |
| SAR 相对 best single-stage 提升 | +0.48 |
| SAR 相对 Standard SFT 提升 | +6.59 |
| Router-side dispatch / sample | 325.5 ms |
| 四 adapter 总存储 | 3.49 GB |

---

## 二、新增实验：多种子稳定性评估

ACM MM 原稿所有结果均为单次运行，在 Limitations 中明确承认 "All reported results are from single training runs with fixed random seeds" 是一个限制。

TMM 稿新增 **3-seed 多随机种子实验（E03）**：
- 种子列表：`[42, 3407, 2025]`
- 对比：Best single-stage `2D-CL` vs `2D-CL+SAR`

**结果**
| 方法 | ChartQA (mean ± std) |
|---|---|
| 2D-CL (best single stage) | 80.24 ± 0.00 |
| 2D-CL+SAR | **82.61 ± 0.04** |

SAR 在三种子下稳定超越 best single-stage，方差仅 ±0.04。

---

## 三、新增实验：路由专项分析

ACM MM 原稿有 "Stage-Specific Expertise" 定性分析，但没有路由实验。

TMM 稿新增 **完整的 Routing Analysis（E05）**：
- 每个 stage 单独在 ChartQA 上的表现
- Oracle routing 上界
- SAR 预测路由
- 最终固定 routing policy（`stage2_fallback + abs_threshold=0.20`）
- 路由分发比例：`stage2=95.0%`, `stage3=4.04%`, `stage4=0.88%`, `stage5=0.08%`

**结果**
| 推理策略 | ChartQA |
|---|---|
| Stage 2 only | 82.12 |
| Stage 3 only | 80.00 |
| Stage 4 only | 82.00 |
| Stage 5 only | 79.32 |
| Oracle routing | 87.48 |
| Predicted routing (SAR) | **82.60** |

Oracle routing 87.48% 揭示了 routeable headroom 巨大，当前 SAR 已经比 best fixed stage 好，但路由策略本身仍有 4.88 点提升空间。

## 五、新增实验：错误分析

ACM MM 原稿有 qualitative examples 但没有系统的错误类型分解。

TMM 稿新增 **错误分析（E10）**：
- 从 ChartQA 全量样本中抽取 300 条共同失败案例
- DeepSeek-V3 API 自动标注 6 类错误
- 建立了标准化的错误分类体系

**结果**（300 条共同失败样本的分布）
| 错误类型 | 占比 |
|---|---|
| 算术错误 | 35.0% |
| 数值提取错误 | 32.3% |
| 密集感知/计数 | 12.0% |
| 图例/坐标轴映射 | 9.7% |
| 多步推理 | 6.0% |
| 答案格式 | 5.0% |

算术 + 数值提取合计 67.3%，说明数值 grounding 仍是主瓶颈；推理和格式错误仅占 11%，说明 CRS 和 staging 已大幅解决这两类问题。

## 六、新增实验：难度可靠性验证

ACM MM 原稿提及 DACDS 但未对难度评分本身做验证。

TMM 稿新增 **难度可靠性分析（E08）**：
- 分析 1973 条合成数据的 `llm_difficulty_level` (1-5)
- 按问题类型分组，验证难度分与数学复杂度单调对齐
- 结合消融实验量化排序本身的影响

**结果**
| 问题类型 | 平均难度 |
|---|---|
| Min/Max lookup | 1.0-1.5 |
| 差值计算 | 2.0 |
| 百分比 | 2.6 |
| 求和 | 2.9 |
| 计数/过滤 | 3.6 |
| 均值 | 3.7 |
| 方差 | 5.0 |

难度等级与数学复杂度完全单调对齐。消融：移除难度排序后 ChartQA 从 82.12% 降到 80.84%（-1.28%）。
